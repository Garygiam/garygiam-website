# PB13.1 — Homepage Evidence Refinement Design

Date: 2026-06-28
Status: Approved in conversation, written for review
Scope: Homepage-visible refinement only

## Objective

Refine the PB13 Homepage Reference Implementation using evidence from the first external-visitor review.

This is not a redesign.

It is a bounded refinement tranche intended to strengthen category creation by making visitors feel why `GEH` must exist before the homepage asks them to accept the category.

## Governing Strategic Conclusion

The remaining homepage gap is not primarily explaining `GEH`.

The remaining gap is helping visitors recognize a recurring organizational reality they already understand, so `GEH` becomes the inevitable conclusion rather than the homepage's opinion.

The homepage should therefore lead visitors through:

1. Observable reality
2. Why current approaches fail
3. Why a new category is necessary
4. `GEH` as the logical solution

## Scope

Included:

- homepage-visible narrative refinements
- homepage-visible proof refinements
- supporting content-model updates required to keep the homepage truthful
- supporting authority-asset or runtime-projection updates required to strengthen homepage proof
- factual integrity corrections needed for internal consistency
- homepage verification, Knowledge Harvest, Promotion Review, and CEO checkpoint closure

Not included:

- About implementation
- Media implementation
- Insights implementation
- Impact implementation
- route expansion
- page redesign
- new public narratives outside the frozen PB13 architecture

## Boundaries

### Homepage-Visible Only

All visitor-facing refinements in PB13.1 must remain limited to the homepage.

Supporting content models, runtime projections, authority assets, and tests may change where necessary, but:

- the `Media` page remains visually and narratively frozen
- the `About` page remains frozen pending future authorization
- the `Insights` page remains frozen pending its dedicated PB13 tranche
- the `Impact` page remains frozen pending its dedicated PB13 tranche

### No Redesign Rule

PB13.1 may refine:

- section purpose
- copy structure
- proof framing
- supporting evidence selection

PB13.1 may not:

- change route structure
- introduce a new homepage architecture
- replace the existing emotional sequence with a different one
- inflate the homepage into a full explanation of the whole system

### Curiosity Preservation Rule

The homepage must still create understanding and curiosity.

It should not attempt to explain every part of `GEH`.

PB13.1 may strengthen necessity and proof, but should not turn the homepage into a manifesto or a case-study archive.

## Evidence Basis

The first external-visitor review established these evidence items:

- the homepage is constitutionally aligned and technically strong
- the main remaining gap is category necessity, not category naming
- visitors need to feel why `GEH` must exist before the category is introduced
- proof should demonstrate organizational strengthening, not merely venture variety
- transformation evidence is stronger than association evidence
- `HMIOSS` must be updated to the current official appointment: `Deputy Chairman`
- `Insights` should later evolve into a `GEH Thinking Center`
- `Impact` should later evolve into a `GEH Outcomes Center`

## PB13.1 Mission

Strengthen the homepage so a first-time visitor recognizes a recurring organizational reality, sees why current approaches fail, understands why a new category is required, and experiences `GEH` as the logical solution.

## Refinement Priorities

## P0 — Category Necessity

### Goal

Strengthen the bridge between:

- observable pain
- why current approaches fail
- why a new category is necessary
- why `GEH` exists

### Narrative Requirement

The homepage should not persuade visitors that `GEH` exists.

Reality should persuade them first.

The homepage should help visitors recognize:

- organizations often gain traction without gaining corresponding strength
- growth often amplifies fragility instead of building capability
- founder effort and opportunistic execution do not reliably create stronger organizations

Only after that recognition should the homepage introduce `GEH` as the necessary category.

### Design Consequence

PB13.1 should refine the opening narrative so the movement from problem to category feels inevitable.

The category should read like the answer to an already-felt reality, not a new label introduced too early.

## P0 — System-Led Proof

### Goal

Reframe the proof layer so the ventures answer:

- What does this prove about `GEH`?
- What organizational strengthening does this prove?

### Narrative Requirement

The proof layer should no longer lean on:

- “what companies has Gary built?”

It should instead show:

- how different expressions of work validate the operating-system logic
- how those expressions demonstrate stronger organizations, stronger capability, or stronger outcomes

### Design Consequence

Each proof element should support the public conclusion that `GEH` helps create stronger organizations rather than simply cataloguing ventures or achievements.

## P1 — Transformation Over Association

### Goal

Expand the public proof inventory using verified transformation evidence.

### Proof Priority Order

Prefer:

- business outcomes
- community outcomes
- leadership outcomes
- venture milestones
- verified public recognition

Do not increase emphasis on:

- VIP photographs
- minister photographs
- association imagery that does not prove strengthening

### Design Consequence

PB13.1 may expand the homepage-accessible proof inventory through supporting authority data and runtime projections, but only where the added evidence demonstrates transformation or strengthening.

## P1 — Factual Integrity

### Goal

Update `HMIOSS` to the current official appointment:

- `Deputy Chairman`

### Consistency Requirement

Runtime content, supporting authority assets where applicable, and tests must remain internally consistent.

PB13.1 should improve trust by tightening factual accuracy while the proof layer is being refined.

## P2 — Future Tranche Preparation

### Goal

Prepare later PB13 direction without implementing those later tranches now.

### Future Direction

- `Insights` should evolve into: `GEH Thinking Center`
- `Impact` should evolve into: `GEH Outcomes Center`

### Boundary

These are planning consequences only for PB13.1.

They should inform later tranches, but no visual or narrative implementation for those routes is authorized here.

## Homepage Narrative Design

PB13.1 should preserve the existing homepage section order while refining the narrative weight inside it.

### Required Narrative Progression

The homepage should still feel like one visitor journey:

- pain
- curiosity
- understanding
- belief
- aspiration
- action

Within that journey, the top half should now more explicitly carry:

1. observable reality
2. failure of current approaches
3. necessity of a new category
4. `GEH` as the logical solution

### Section-Level Refinement Intent

#### Hero / Problem Entry

Should sharpen recognizable organizational reality.

It should make visitors feel:

- this problem already exists
- they have seen it before
- it is not a rhetorical setup

#### Category Introduction

Should arrive after the necessity bridge is stronger.

It should feel like:

- naming the solution to an already-recognized problem

not:

- asking visitors to accept a founder-defined idea on faith

#### Operating-System Layer

Should continue to explain why `GEH` is different, but now in direct continuity with the “why current approaches fail” bridge.

#### Proof Layer

Should demonstrate:

- that the system is being expressed across ventures
- that the expressions produce visible organizational strengthening

#### Future / Action Layer

Should continue to make engagement feel like participation in a meaningful system rather than generic founder outreach.

## Homepage Proof Design

PB13.1 proof refinement should follow these rules:

### Proof Standard

Every proof element should support both of these conclusions:

- `GEH` is real enough to validate publicly
- `GEH` helps create stronger organizations

### Proof Classes Allowed In This Tranche

- venture-expression proof already present in the homepage ecosystem layer
- verified authority proof already present in the authority vault
- verified milestone proof already present in the authority vault
- verified media proof already present in the authority vault

### Proof Classes Not To Prioritize

- status signaling without strengthening evidence
- prestige-by-association
- social proximity proof that does not show organizational transformation
