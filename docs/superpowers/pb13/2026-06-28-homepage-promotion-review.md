# PB13 Homepage Promotion Review

Date: 2026-06-28
Programme: PB13 — GEH Public Narrative Implementation
Tranche: 1 — Homepage

## Review Purpose

This review does not modify GEH Narrative Architecture v1.

It classifies implementation discoveries as:

- Homepage-local only
- Reusable across GaryGiam.com
- Future GEH pattern candidate

## Items

### Homepage Problem Frame

- Classification: `Reusable across GaryGiam.com`
- Evidence: the homepage became clearer once it opened with a recurring organizational pain instead of beginning with founder credentials or venture inventory.
- Reuse Decision: carry this problem-first entry pattern into `About`, `Companies`, `Media`, `Insights`, and `Contact`, adapting the proof depth to each page without changing the constitutional narrative order.

### Homepage Category Definition Block

- Classification: `Future GEH pattern candidate`
- Evidence: the dedicated category section gave first-time visitors a single public-language definition of `GEH`, improving the homepage's 30-second message discipline.
- Reuse Decision: reuse the category-definition pattern across GaryGiam.com now, and evaluate later whether it should become a default public-category pattern for future GEH instances.

### Homepage Operating-System Principle Layer

- Classification: `Future GEH pattern candidate`
- Evidence: the canonical definition plus three principles created understandable differentiation without over-explaining the system, which matched the approved curiosity-over-completeness rule.
- Reuse Decision: preserve this pattern for future high-level system explanation pages and assess cross-instance evidence before promoting it beyond GaryGiam.com.

### Homepage Proof Through Venture Expression

- Classification: `Reusable across GaryGiam.com`
- Evidence: ventures worked best when positioned as expressions of the operating system, not as unrelated company cards or a founder portfolio.
- Reuse Decision: carry this proof model into `Companies` and `Media / Authority`, where each entity can be expanded without losing the system-first framing.

### Homepage Future / Action Framework

- Classification: `Reusable across GaryGiam.com`
- Evidence: distinct CTAs tied to understanding, proof, and partnership reinforced the emotional progression from aspiration to action and prevented duplicated CTA ambiguity.
- Reuse Decision: keep intent-specific CTA naming as a site-wide rule and extend the pattern into `Contact` and later conversion-oriented surfaces.
